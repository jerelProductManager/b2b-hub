Problem Space

Why the Current Hub Isn't Working

📄
Content Is Locked in Code
All updates to a headline, logo, and contract listing requires a Jira ticket, dev time, QA, and a deployment cycle. Even a logo update can take months. Insufficient GA4 tagging obscures customer engagement analysis.
→
Sign-Up Is Buried
Users must browse segments and sub-pages before reaching a registration option. No organizational, state, or account info is required upfront, leading to erroneous routing and wasted rep time.
📍
Routing Logic Is Hard-Coded; Verification Impeded
State-to-contract-to-portal routing is embedded in the codebase. Adding a new state, contract, or portal association requires a developer. No self-service path exists.
⚙️
Weblogic Constrains Everything
The legacy platform makes CMS extensibility impractical. Every architectural improvement is bottlenecked by platform limitations that Aperture is designed to remove.
03 / 07
Updated Scope

Three Strategic Pivots

What is concretely changing in Phase 1.

01
Weblogic → Aperture
All deliverables built on Aperture. No new work on Weblogic. Platform migration and hub modernization are coordinated, not sequential. All elements GA4 tagged at launch.
02
Segment Browsing → Sign-Up First
Sign-up is the hero CTA. Qualification-first funnel collects org type, state, and contact info before surfacing cooperative contracts or portal options.
03
Hard-Coded → JSON-Configurable Content & Routing
All hub, segment, and contract page content driven by JSON config, enabling the B2B team to publish changes without developer involvement. Contract × segment × state routing logic is also moved to JSON-configurable — new associations added via JSON config, no code change required.

Replacing Hard-Code with Configurable JSON

Every content field and routing rule becomes editable — no dev ticket required.

Body copy + benefit bullets

Add / update without code change
Contract × Segment × State Routing
State → cooperative contract mapping
Segment → portal routing rules
New associations via JSON config

Constraints

Constraints the Team Needs to Navigate

Known gates and open decisions that affect sequencing.

⛔ Manny Steigman Approval Gate
Customer-facing changes require Manny Steigman's sign-off before going live. Manny is unavailable until approximately May 2026. Phase 1 build can proceed in parallel — launch is gated on this approval.
⚠️ Aperture Migration Sequencing
Phase 1 must be built on Aperture. Any work delivered on Weblogic first creates porting liability. Migration timeline alignment is open question #1 for the architect session.
⚠️ PMO 5000 — Existing-Customer Routing
Routing logic for existing B2B accounts is architecturally unresolved. Current lean: apply new routing logic only to new sign-ups, leave existing accounts unchanged. Not formally closed.
